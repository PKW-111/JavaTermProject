# 2. 요구사항 분석 및 설계

## 1. 요구사항

FreshKeeper Smart Fridge는 사용자가 직접 입력한 냉장고 식재료 정보를 체계적으로 관리하고, 유통기한이 임박한 식재료를 우선적으로 소비할 수 있도록 돕는 Java 데스크톱 GUI 프로그램이다.

사용자는 Java Swing 기반 화면에서 식재료를 등록하고, 전체 목록을 조회하며, 검색·수정·삭제할 수 있다. 프로그램은 현재 날짜를 기준으로 유통기한까지 남은 일수를 계산하고, 식재료를 만료, 오늘까지, 위험, 주의, 안전 상태로 분류한다.

또한 보유 중인 식재료와 레시피를 비교하여 메뉴를 추천하고, 부족한 재료와 추천 점수를 표시한다. 추천 메뉴를 조리하면 사용된 식재료의 수량이 자동으로 차감되며, 식재료 데이터는 CSV 파일에 저장되어 프로그램을 다시 실행해도 유지된다.

## 1.3 요구사항 정리

본 프로젝트의 핵심 요구사항은 사용자가 직접 식재료 정보를 등록하고, 등록된 식재료의 수량과 유통기한 상태를 쉽게 확인할 수 있도록 하는 것이다.

초기에는 콘솔 기반 메뉴를 통해 식재료를 관리하도록 설계하였으나, 사용성과 시각적 완성도를 높이기 위해 Java Swing 기반 데스크톱 GUI 프로그램으로 확장하였다.

사용자는 GUI의 입력창과 버튼을 이용하여 식재료를 등록, 검색, 수정, 삭제할 수 있다. 등록된 식재료는 테이블 형태로 표시되며, 프로그램은 현재 날짜를 기준으로 유통기한 남은 일수와 상태를 자동으로 계산한다.

또한 보유 식재료를 기반으로 메뉴를 추천하고, 부족한 재료와 추천 점수를 표시한다. 추천 메뉴를 조리하면 사용된 식재료의 수량이 자동으로 차감되어 식재료 관리와 메뉴 추천 기능이 하나의 흐름으로 연결된다.

식재료 데이터는 CSV 파일에 저장되므로 프로그램을 종료한 뒤 다시 실행하더라도 기존 데이터가 유지된다.

다만 본 프로젝트는 다음 기능을 포함하지 않는다.

* 실제 냉장고 내부 자동 스캔
* 바코드 인식
* 카메라 이미지 인식
* IoT 냉장고 센서 연동
* 외부 레시피 API 연동
* 웹 서버 및 클라우드 동기화
* 모바일 앱 개발
* 회원가입 및 로그인
* 데이터베이스 연동

본 프로젝트는 Java 표준 라이브러리와 Java Swing을 이용하여 구현 가능한 식재료 관리, 유통기한 계산, 메뉴 추천, 파일 저장 기능에 집중하였다.

# 시스템 설계

## 2. 전체 프로젝트 개요

FreshKeeper Smart Fridge는 사용자가 Java Swing GUI에서 식재료 정보를 입력하면, 서비스 클래스가 데이터를 관리하고 유통기한을 계산한 뒤 메뉴 추천 결과를 화면에 표시하는 구조이다.

식재료 데이터는 실행 중에는 `ArrayList`로 관리되며, 프로그램 종료 시 CSV 파일에 저장된다.

---

## 2.1 전체 처리 흐름

| 단계 | 담당 요소               | 역할                                     |
| -- | ------------------- | -------------------------------------- |
| 1  | 사용자                 | GUI에서 식재료 입력, 기능 버튼 선택                 |
| 2  | Main.java           | 프로그램 시작 및 GUI 실행                       |
| 3  | FreshKeeperGUI.java | 입력창, 테이블, 버튼, 상태 카드, 추천 화면 구성 및 이벤트 처리 |
| 4  | FoodService.java    | 식재료 등록, 조회, 검색, 수정, 삭제, CSV 저장 및 불러오기  |
| 5  | Food.java           | 식재료 정보 저장, 유통기한 계산, 상태 분류              |
| 6  | RecipeService.java  | 보유 재료와 레시피 비교, 추천 점수 계산                |
| 7  | Recipe.java         | 메뉴 이름과 필요한 재료 목록 저장                    |
| 8  | RecipeResult.java   | 추천 결과, 부족 재료, 추천 점수, 조리 처리             |
| 9  | foods.csv           | 프로그램 종료 후 식재료 데이터 저장                   |

# 3. 자바 클래스 설계

본 프로젝트는 화면 처리, 데이터 관리, 메뉴 추천 기능을 클래스별로 분리하여 유지보수성과 확장성을 높였다.

| 클래스명                | 구분         | 주요 역할                                          |
| ------------------- | ---------- | ---------------------------------------------- |
| Main.java           | 실행 클래스     | 프로그램 시작 및 FreshKeeper GUI 실행                   |
| FreshKeeperGUI.java | 화면 클래스     | Swing 화면 구성, 사용자 입력 처리, 버튼 이벤트, 테이블 및 상태 카드 갱신 |
| Food.java           | 모델 클래스     | 식재료 정보 저장, 유통기한 남은 일수 계산, 상태 분류                |

| RecipeService.java  | 서비스 클래스    | 보유 재료 비교, 부족 재료 확인, 추천 점수 계산                   |

## 3.3 주요 클래스 동작

### Main.java

* 프로그램의 시작점이다.
* `SwingUtilities.invokeLater()`를 이용하여 GUI를 실행한다.

### FreshKeeperGUI.java

* 식재료 입력창과 기능 버튼을 표시한다.
* 식재료 목록을 `JTable`에 표시한다.
* 상단 상태 카드에 만료, 위험, 주의, 안전 개수를 표시한다.
* 메뉴 추천 결과를 카드 형태로 출력한다.
* 등록, 검색, 수정, 삭제, 조리, 저장 이벤트를 처리한다.

### Food.java

* 식재료 ID, 이름, 카테고리, 수량, 위치, 유통기한을 저장한다.
* 현재 날짜를 기준으로 남은 유통기한을 계산한다.
* 남은 일수를 기준으로 식재료 상태를 반환한다.

### FoodService.java

* `ArrayList<Food>`를 이용해 식재료 목록을 관리한다.
* 식재료 등록, 조회, 검색, 수정, 삭제 기능을 처리한다.
* CSV 파일에 데이터를 저장하고 불러온다.

### RecipeService.java

* 보유 재료와 레시피 재료를 비교한다.
* 보유 재료, 부족 재료, 임박 재료를 바탕으로 추천 점수를 계산한다.
* 점수가 높은 메뉴를 우선적으로 추천한다.

### RecipeResult.java

* 추천된 메뉴와 추천 점수를 저장한다.
* 보유 재료와 부족 재료 정보를 저장한다.
* 메뉴 조리 시 사용된 식재료의 수량을 차감한다.

# 4. 데이터 저장에 필요한 필드 정리

본 프로젝트는 별도의 데이터베이스를 사용하지 않고, 실행 중에는 Java의 `ArrayList`를 이용하여 식재료 데이터를 관리한다.

프로그램을 종료한 후에도 데이터가 유지되도록 CSV 파일에 식재료 정보를 저장한다.

## 4.1 식재료 데이터 필드

| 구분    | 필드명           | 자료형         | 설명                      |
| ----- | ------------- | ----------- | ----------------------- |
| 저장 필드 | id            | int 또는 long | 식재료 고유 번호               |
| 저장 필드 | name          | String      | 식재료 이름                  |
| 저장 필드 | category      | String      | 식재료 카테고리                |
| 저장 필드 | quantity      | int         | 보유 수량                   |
| 저장 필드 | location      | String      | 냉장, 냉동, 실온 등의 보관 위치     |
| 저장 필드 | expiryDate    | LocalDate   | 식재료 유통기한                |
| 계산 필드 | daysRemaining | long        | 현재 날짜 기준 남은 유통기한 일수     |
| 계산 필드 | status        | String      | 만료, 오늘까지, 위험, 주의, 안전 상태 |

`daysRemaining`과 `status`는 유통기한을 기준으로 프로그램 실행 중 계산되는 값이므로 CSV 파일에 반드시 저장할 필요는 없다.

---

## 4.2 CSV 저장 구조

식재료 데이터는 다음 형식으로 저장할 수 있다.

```text
id,name,category,quantity,location,expiryDate
1,계란,단백질,10,냉장,2026-06-20
2,닭가슴살,단백질,5,냉동,2030-01-01
```

| CSV 순서 | 필드명        | 예시         |
| ------ | ---------- | ---------- |
| 1      | id         | 1          |
| 2      | name       | 계란         |
| 3      | category   | 단백질        |
| 4      | quantity   | 10         |
| 5      | location   | 냉장         |
| 6      | expiryDate | 2026-06-20 |

---

## 4.3 레시피 데이터 필드

| 필드명         | 자료형          | 설명               |
| ----------- | ------------ | ---------------- |
| name        | String       | 메뉴 이름            |
| ingredients | List<String> | 메뉴 조리에 필요한 재료 목록 |

---

## 4.4 추천 결과 필드

| 필드명                | 자료형          | 설명                     |
| ------------------ | ------------ | ---------------------- |
| recipe             | Recipe       | 추천된 메뉴 정보              |
| matchedIngredients | List<String> | 현재 보유하고 있는 재료          |
| missingIngredients | List<String> | 현재 보유하지 않은 부족 재료       |
| matchScore         | int          | 보유 재료와 유통기한을 반영한 추천 점수 |

# 5. 화면 설계

FreshKeeper Smart Fridge는 Java Swing 기반 데스크톱 GUI 프로그램으로 구성한다.

## 5.1 화면 구성

| 화면 영역 | 구성 요소             | 역할                              |
| ----- | ----------------- | ------------------------------- |
| 상단    | 프로그램 제목, 상태 요약 카드 | 만료, 위험, 주의, 안전 식재료 개수 표시        |
| 왼쪽    | 입력창, 검색창          | 식재료 등록, 수정, 검색 정보 입력            |
| 가운데   | JTable            | 등록된 식재료 정보와 상태 표시               |
| 오른쪽   | 메뉴 추천 카드          | 메뉴명, 추천 점수, 보유 및 부족 재료 표시       |
| 하단    | 기능 버튼             | 등록, 수정, 삭제, 검색, 초기화, 조리, 저장, 종료 |

---

## 5.2 입력 항목

| 입력 항목  | 입력 예시      |
| ------ | ---------- |
| 식재료 이름 | 계란         |
| 카테고리   | 단백질        |
| 수량     | 10         |
| 보관 위치  | 냉장         |
| 유통기한   | 2026-06-20 |
| 검색어    | 계란 또는 냉장   |

---

## 5.3 화면 처리 흐름

```text
프로그램 실행
  ↓
CSV 데이터 불러오기
  ↓
GUI 화면 출력
  ↓
식재료 등록·검색·수정·삭제
  ↓
테이블 및 상태 카드 갱신
  ↓
메뉴 추천 확인
  ↓
추천 메뉴 조리
  ↓
식재료 수량 차감
  ↓
CSV 저장 후 종료
```
