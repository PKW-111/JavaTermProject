# 1. 프로젝트 개요

## 1.1 프로젝트명

**FreshKeeper: 사용자 입력 기반 냉장고 유통기한 관리 및 메뉴 추천 프로그램**

---

## 1.2 프로젝트 소개

FreshKeeper는 사용자가 직접 입력한 식재료 정보를 기반으로 냉장고 속 식재료를 관리하는 Java 콘솔 프로그램이다.

사용자는 식재료의 이름, 카테고리, 수량, 보관 위치, 유통기한을 입력할 수 있으며, 프로그램은 현재 날짜를 기준으로 유통기한까지 남은 날짜를 계산한다. 계산된 결과에 따라 식재료를 안전, 주의, 위험, 오늘까지, 만료 상태로 분류하여 사용자가 어떤 재료를 먼저 사용해야 하는지 쉽게 확인할 수 있도록 한다.

또한 등록된 식재료와 레시피 데이터를 비교하여 만들 수 있는 메뉴를 추천하고, 부족한 재료를 함께 표시함으로써 식재료 활용도를 높이는 것을 목표로 한다.

---

## 1.3 프로젝트 목적

본 프로젝트의 목적은 냉장고 속 식재료를 효율적으로 관리하고, 유통기한이 임박한 재료를 우선적으로 사용할 수 있도록 돕는 것이다.

일상생활에서 사용자는 냉장고에 어떤 재료가 있는지 잊어버리거나, 유통기한을 확인하지 못해 식재료를 버리는 경우가 있다. FreshKeeper는 이러한 문제를 줄이기 위해 사용자가 구매한 식재료를 직접 등록하고, 전체 조회를 통해 보유 재료와 유통기한 상태를 확인할 수 있도록 한다.

이를 통해 음식물 낭비를 줄이고, 보유 중인 재료를 활용한 메뉴 선택에도 도움을 주고자 한다.

---

## 1.4 개발 배경

현대 사회에서는 자취생이나 바쁜 대학생처럼 직접 식재료를 구매하고 관리해야 하는 사람들이 많다. 그러나 냉장고 속 재료를 매번 직접 확인하는 것은 번거롭고, 유통기한이 짧은 재료를 제때 사용하지 못하는 경우가 자주 발생한다.

기존 방식은 냉장고를 직접 열어 확인하거나 메모장에 식재료를 기록하는 방식이 대부분이다. 하지만 이러한 방법은 유통기한이 얼마나 남았는지 한눈에 파악하기 어렵고, 보유한 재료로 어떤 음식을 만들 수 있는지 판단하기 어렵다는 한계가 있다.

따라서 본 프로젝트에서는 사용자가 직접 입력한 식재료 정보를 바탕으로 유통기한을 자동 계산하고, 보유 재료 기반 메뉴 추천 기능을 제공하는 프로그램을 개발하였다.

---

## 1.5 대상 사용자

- 냉장고 속 식재료를 체계적으로 관리하고 싶은 사용자
- 자취를 하며 식재료를 직접 관리하는 대학생
- 유통기한이 임박한 재료를 먼저 사용하고 싶은 사용자
- 보유 재료로 만들 수 있는 메뉴를 추천받고 싶은 사용자
- 같은 식재료를 중복 구매하거나 유통기한을 놓치는 일을 줄이고 싶은 사용자

---

## 1.6 기존 방법 대비 개선점

| 기존 방법 | 문제점 | FreshKeeper의 개선점 |
|---|---|---|
| 냉장고를 직접 확인 | 어떤 재료가 있는지 매번 확인해야 함 | 등록된 식재료를 전체 조회로 확인 가능 |
| 메모장에 직접 기록 | 유통기한 계산을 직접 해야 함 | 현재 날짜 기준 남은 날짜 자동 계산 |
| 기억에 의존 | 유통기한을 잊어버리기 쉬움 | 안전/주의/위험/오늘까지/만료 상태 표시 |
| 보유 재료로 메뉴를 직접 고민 | 만들 수 있는 메뉴를 바로 알기 어려움 | 보유 재료 기반 메뉴 추천 |
| 부족 재료 확인 어려움 | 장보기 계획이 불편함 | 추천 메뉴의 부족 재료 표시 |

## 1.7 주요 기능

| 기능 | 설명 |
|---|---|
| 식재료 등록 | 식재료 이름, 카테고리, 수량, 보관 위치, 유통기한을 입력한다. |
| 식재료 전체 조회 | 등록된 식재료 목록을 확인한다. |
| 식재료 삭제 | ID를 기준으로 등록된 식재료를 삭제한다. |
| 유통기한 계산 | 현재 날짜를 기준으로 유통기한까지 남은 일수를 계산한다. |
| 위험도 분류 | 남은 날짜에 따라 안전, 주의, 위험, 오늘까지, 만료 상태로 분류한다. |
| 메뉴 추천 | 보유 중인 식재료를 기반으로 만들 수 있는 메뉴를 추천한다. |
| 부족 재료 표시 | 추천 메뉴에 필요한 재료 중 부족한 재료를 표시한다. |

## 1.8 프로젝트 범위

### 포함 범위

- 사용자가 직접 식재료 정보 입력
- 식재료 등록, 조회, 삭제
- 유통기한 남은 날짜 계산
- 안전, 주의, 위험, 오늘까지, 만료 상태 분류
- 보유 재료 기반 메뉴 추천
- 부족한 재료 표시
- Java 콘솔 기반 프로그램 구현

### 제외 범위

- 실제 냉장고 내부 자동 스캔
- 바코드 스캔
- 이미지 인식
- IoT 냉장고 센서 연동
- 외부 레시피 API 연동
- 모바일 앱 개발
- 회원가입 및 로그인 기능
- 데이터베이스 연동

본 프로젝트는 실제 냉장고를 자동으로 스캔하는 시스템이 아니라, 사용자가 구매한 식재료를 직접 입력하고 관리하는 사용자 입력 기반 프로그램이다.

## 1.9 기대 효과

- 냉장고 속 식재료를 체계적으로 관리할 수 있다.
- 유통기한이 임박한 식재료를 쉽게 확인할 수 있다.
- 어떤 재료를 먼저 사용해야 하는지 판단할 수 있다.
- 보유 재료 기반 메뉴 추천을 통해 식재료 활용도를 높일 수 있다.
- 부족한 재료를 확인하여 장보기 계획에 도움을 줄 수 있다.
- 음식물 낭비를 줄이는 데 도움을 줄 수 있다.
- Java의 클래스, 객체, ArrayList, 반복문, 예외처리, 날짜 계산 개념을 실제 프로젝트에 적용할 수 있다.
