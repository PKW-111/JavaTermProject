# 4. 기능 구현 및 단계별 구현 내용

FreshKeeper Smart Fridge는 사용자가 직접 입력한 식재료 정보를 관리하고, 유통기한 상태를 확인하며, 보유 재료를 기반으로 메뉴를 추천하는 Java 데스크톱 GUI 프로그램이다.

초기에는 콘솔 환경에서 핵심 기능을 먼저 구현하였으며, 이후 사용 편의성과 프로젝트 완성도를 높이기 위해 Java Swing 기반의 스마트 냉장고 형태 GUI로 확장하였다.

본 문서는 프로젝트 개요와 요구사항 분석을 바탕으로 실제 구현한 기능을 기능별·단계별로 정리한 것이다.

---

## 4.2 단계별 구현 내용

### 1단계: 콘솔 기반 기본 구조 구현

프로젝트 초기에는 핵심 기능을 빠르게 확인하기 위해 콘솔 기반으로 구현하였다.

`Main` 클래스를 중심으로 반복문과 조건문을 이용하여 메뉴 선택 흐름을 구성하였으며, 사용자는 번호를 입력하여 식재료 등록, 조회, 삭제, 메뉴 추천, 종료 기능을 실행할 수 있도록 하였다.

이 단계에서는 전체 프로그램의 기본 동작과 클래스 간 연결을 확인하는 데 중점을 두었다.

---

### 2단계: 식재료 데이터 모델 구현

`Food` 클래스를 이용하여 식재료의 정보를 객체로 관리하도록 구현하였다.

`Food` 클래스의 주요 필드는 다음과 같다.

| 필드           | 설명        |
| ------------ | --------- |
| `id`         | 식재료 고유 번호 |
| `name`       | 식재료 이름    |
| `category`   | 식재료 카테고리  |
| `quantity`   | 식재료 보유 수량 |
| `location`   | 보관 위치     |
| `expiryDate` | 유통기한      |

`getDaysRemaining()` 또는 동일한 역할의 메서드를 이용하여 현재 날짜와 유통기한 사이의 남은 일수를 계산하였다.

`getStatus()` 메서드는 계산된 남은 일수를 바탕으로 식재료 상태를 반환하도록 구현하였다.

---

### 3단계: 식재료 등록·조회·삭제 기능 구현

`FoodService` 클래스에서 `ArrayList<Food>`를 이용하여 식재료 목록을 관리하였다.

식재료 등록 시에는 새로운 `Food` 객체를 생성하고, 중복되지 않는 ID를 자동으로 부여하여 목록에 추가하였다.

전체 조회 기능은 등록된 식재료의 다음 정보를 한 번에 확인할 수 있도록 구현하였다.

* ID
* 이름
* 카테고리
* 수량
* 보관 위치
* 유통기한
* 남은 일수
* 상태

삭제 기능은 식재료 ID 또는 GUI 테이블에서 선택한 항목을 기준으로 실행되며, 실수로 삭제하는 것을 방지하기 위해 삭제 확인창을 표시하도록 구현하였다.

---

### 4단계: 유통기한 계산 및 상태 분류 구현

`Food` 클래스에서 `LocalDate`를 이용하여 현재 날짜와 유통기한을 비교하였다.

유통기한 상태 기준은 다음과 같다.

| 남은 일수 | 상태   |
| ----- | ---- |
| 0일 미만 | 만료   |
| 0일    | 오늘까지 |
| 1~2일  | 위험   |
| 3~6일  | 주의   |
| 7일 이상 | 안전   |

상태 분류 결과는 식재료 테이블, 유통기한 알림, 상태 요약 카드, 메뉴 추천 점수 계산에서 공통으로 사용한다.

---

### 5단계: 입력 예외 처리 구현

사용자가 잘못된 값을 입력해도 프로그램이 비정상적으로 종료되지 않도록 입력 검증 기능을 구현하였다.

주요 예외 처리 내용은 다음과 같다.

* 수량에 문자를 입력한 경우 숫자 입력 안내
* 수량이 음수이거나 0 이하인 경우 재입력 안내
* 날짜 형식이 `yyyy-MM-dd`가 아닌 경우 오류 안내
* 필수 입력칸이 비어 있는 경우 입력 요청
* 테이블에서 항목을 선택하지 않고 수정 또는 삭제한 경우 선택 안내
* 검색어를 입력하지 않고 검색한 경우 입력 요청

오류가 발생하면 `JOptionPane`을 이용하여 사용자에게 원인과 올바른 입력 예시를 표시한다.

---

### 6단계: CSV 저장 및 불러오기 기능 구현

프로그램 종료 후에도 식재료 데이터가 유지되도록 CSV 파일 기반 저장 기능을 구현하였다.

프로그램을 실행하면 `data/foods.csv` 파일이 존재하는지 확인하고, 파일이 있으면 기존 식재료 데이터를 자동으로 불러온다.

저장 버튼을 누르거나 프로그램을 종료하면 현재 식재료 목록을 CSV 파일에 저장한다.

저장되는 데이터는 다음과 같다.

```text
id,name,category,quantity,location,expiryDate
```

남은 일수와 상태는 유통기한을 기준으로 실행 중 자동 계산할 수 있으므로 CSV 파일에는 별도로 저장하지 않는다.

---

### 7단계: 식재료 검색 및 수정 기능 구현

기존 등록·조회·삭제 기능에 검색 및 수정 기능을 추가하였다.

검색 기능은 다음 항목을 기준으로 식재료를 찾을 수 있도록 구현하였다.

* 식재료 이름
* 카테고리
* 보관 위치

검색 결과에 같은 식재료가 여러 조건으로 중복 포함되지 않도록 ID를 기준으로 중복을 제거하였다.

수정 기능은 테이블에서 식재료를 선택한 뒤 입력창에 변경할 정보를 작성하고 수정 버튼을 누르면 해당 객체의 정보가 변경되도록 구현하였다.

수정 후에는 테이블과 상태 요약 카드가 다시 갱신되도록 구성하였다.

---

### 8단계: 레시피 및 메뉴 추천 기능 구현

`Recipe` 클래스는 메뉴 이름과 필요한 재료 목록을 저장한다.

`RecipeService` 클래스에서는 현재 보유 중인 식재료와 각 레시피의 필요 재료를 비교하여 메뉴별 추천 점수를 계산한다.

`RecipeResult` 클래스에는 다음 정보가 저장된다.

* 추천된 메뉴
* 보유 재료
* 부족 재료
* 추천 점수

계산된 추천 결과는 점수가 높은 순서대로 정렬하여 사용자에게 표시한다.

---

### 9단계: 추천 점수 고도화

단순히 보유 재료의 개수만 비교하지 않고 유통기한 상태까지 고려하도록 추천 점수 계산 기능을 개선하였다.

추천 점수 기준은 다음과 같다.

| 조건                   | 점수         |
| -------------------- | ---------- |
| 필요한 재료를 보유한 경우       | 가점         |
| 유통기한이 임박한 재료가 포함된 경우 | 추가 가점      |
| 필요한 재료가 부족한 경우       | 감점         |
| 모든 재료를 보유한 경우        | 추가 가점      |
| 만료된 재료               | 추천 재료에서 제외 |

이를 통해 단순히 만들 수 있는 메뉴가 아니라, 유통기한이 임박한 식재료를 우선적으로 활용할 수 있는 메뉴를 추천하도록 구현하였다.

---

### 10단계: 메뉴 조리 및 수량 차감 기능 구현

추천 메뉴 중 하나를 선택하여 실제로 조리하는 기능을 추가하였다.

사용자가 조리할 메뉴를 선택하면 해당 레시피에 필요한 재료를 확인하고, 보유 중인 식재료의 수량을 자동으로 차감한다.

조리 후 수량이 0 이하가 된 식재료가 있는 경우 다음과 같은 확인 절차를 진행한다.

```text
수량이 0이 된 식재료가 있습니다.
목록에서 삭제하시겠습니까?
```

사용자가 삭제를 선택하면 해당 식재료를 목록에서 제거하며, 삭제하지 않으면 수량 0 상태로 유지한다.

조리 완료 후에는 식재료 테이블, 메뉴 추천 결과, 상태 카드가 다시 갱신된다.

---

### 11단계: Java Swing GUI 구현

초기 콘솔 프로그램을 Java Swing 기반 데스크톱 GUI 프로그램으로 확장하였다.

GUI는 다음 영역으로 구성하였다.

| 화면 영역 | 기능                                 |
| ----- | ---------------------------------- |
| 상단    | 프로그램 제목과 상태 요약 카드 표시               |
| 왼쪽    | 식재료 등록·수정 입력창과 검색창                 |
| 가운데   | 등록된 식재료 목록을 `JTable`로 표시           |
| 오른쪽   | 추천 메뉴와 추천 점수를 카드 형태로 표시            |
| 하단    | 등록, 수정, 삭제, 검색, 초기화, 조리, 저장, 종료 버튼 |

어두운 배경과 네온 계열의 색상을 적용하여 스마트 냉장고 디스플레이와 유사한 형태로 구성하였다.

---

### 12단계: GUI 사용성 개선

사용자가 각 입력칸에 어떤 내용을 작성해야 하는지 쉽게 알 수 있도록 입력 예시 문구를 추가하였다.

예시는 다음과 같다.

```text
식재료 이름 예: 계란
카테고리 예: 단백질, 채소, 반찬
수량 예: 10
보관 위치 예: 냉장, 냉동, 실온
유통기한 예: 2026-06-20
검색어 예: 계란 또는 냉장
```

입력창에 마우스를 올리면 추가 설명이 표시되도록 툴팁을 적용하였다.

한글이 사각형으로 깨지는 문제를 해결하기 위해 Swing 컴포넌트에 한글을 지원하는 폰트를 적용하였다.

또한 일부 환경에서 표시되지 않는 이모지를 제거하고 텍스트 형태의 안내 문구로 변경하였다.

---

### 13단계: 상태 요약 카드 및 알림 구현

프로그램 상단에 다음 상태별 식재료 개수를 표시하는 요약 카드를 구현하였다.

* 만료
* 위험
* 주의
* 안전

식재료 목록을 순회하면서 각 식재료의 `getStatus()` 결과를 확인하고 상태별 개수를 계산한다.

등록, 수정, 삭제, 메뉴 조리, CSV 데이터 불러오기 후에는 상태 개수를 다시 계산하여 카드 숫자를 갱신하도록 구현한다.

프로그램 실행 시 만료 또는 위험 상태의 식재료가 존재하면 알림창을 표시하여 사용자가 먼저 확인할 수 있도록 하였다.

> 상태 카드 숫자 자동 갱신 기능을 수정 중인 경우 구현 상태 문서에는 `🔶 부분완성`으로 표시하고, 문제 해결 후 `✅ 완성`으로 변경한다.

## 4.3 기능별 구현 포인트

### 식재료 등록

* 입력값을 검증한 뒤 `Food` 객체를 생성한다.
* 생성된 객체를 `FoodService`의 목록에 추가한다.
* 등록 후 입력칸을 초기화한다.
* 테이블과 상태 카드, 메뉴 추천 목록을 갱신한다.

### 식재료 조회

* 식재료 정보를 `JTable` 형식으로 표시한다.
* ID, 이름, 카테고리, 수량, 위치, 유통기한, 남은 일수, 상태를 함께 출력한다.
* 테이블 셀은 사용자가 직접 수정하지 못하도록 설정한다.

### 식재료 검색

* 이름, 카테고리, 보관 위치를 기준으로 검색한다.
* 여러 검색 조건에서 같은 식재료가 중복된 경우 ID를 기준으로 하나만 표시한다.
* 초기화 버튼을 누르면 전체 식재료 목록으로 돌아간다.

### 식재료 수정

* 사용자가 테이블에서 수정할 식재료를 선택한다.
* 입력값을 검증한 뒤 해당 ID의 식재료 정보를 변경한다.
* 수정 후 테이블과 상태 카드를 갱신한다.

### 식재료 삭제

* 삭제 전에 선택한 식재료 이름을 확인창에 표시한다.
* 사용자가 확인한 경우에만 목록에서 삭제한다.
* 삭제 후 CSV 저장 대상과 화면 목록을 갱신한다.

### 유통기한 상태 분류

| 남은 일수 | 상태   |
| ----- | ---- |
| 0일 미만 | 만료   |
| 0일    | 오늘까지 |
| 1~2일  | 위험   |
| 3~6일  | 주의   |
| 7일 이상 | 안전   |

상태 계산은 `Food` 클래스에서 담당하며, 다른 클래스는 해당 결과를 사용한다.

### 상태 요약 카드

* 모든 식재료의 상태를 순회하여 상태별 개수를 계산한다.
* 계산된 개수를 상단의 `JLabel`에 표시한다.
* 식재료 목록이 변경될 때마다 다시 계산한다.

### 메뉴 추천

* 보유 식재료와 레시피의 필요 재료를 비교한다.
* 만료된 식재료는 추천 계산에서 제외한다.
* 보유 재료, 부족 재료, 임박 재료 여부를 추천 점수에 반영한다.
* 높은 점수의 메뉴를 우선적으로 표시한다.

### 메뉴 조리

* 추천 메뉴 목록에서 하나를 선택한다.
* 레시피에 필요한 식재료의 수량을 차감한다.
* 수량이 0 이하인 식재료는 삭제 여부를 사용자에게 확인한다.
* 조리 후 전체 화면을 다시 갱신한다.

### CSV 저장 및 불러오기

* 실행 시 CSV 파일에서 식재료 데이터를 불러온다.
* 저장 버튼 또는 프로그램 종료 시 현재 데이터를 저장한다.
* 저장 폴더가 없는 경우 자동으로 생성하도록 처리한다.
* 데이터가 없는 경우 빈 식재료 목록으로 프로그램을 시작한다.

### 입력 예외 처리

* 수량은 정수만 허용한다.
* 유통기한은 `yyyy-MM-dd` 형식만 허용한다.
* 필수 입력값이 누락된 경우 등록 또는 수정을 실행하지 않는다.
* 오류 발생 시 프로그램 종료 대신 안내창을 표시한다.

## 4.5 구현 결과 요약

FreshKeeper Smart Fridge는 식재료 등록, 조회, 검색, 수정, 삭제, 유통기한 계산, 상태 분류, 메뉴 추천, 부족 재료 표시, CSV 저장 및 불러오기 기능을 하나의 Java 데스크톱 프로그램으로 통합하였다.

초기 콘솔 버전에서 Java Swing GUI 버전으로 확장하여 사용자가 입력창과 버튼을 통해 기능을 직관적으로 사용할 수 있도록 하였다.

유통기한 상태와 추천 점수를 연결하여 임박한 식재료를 우선적으로 활용할 수 있도록 하였으며, 추천 메뉴 조리 후 식재료 수량이 자동으로 차감되는 기능을 통해 실제 냉장고 재고 관리 흐름을 구현하였다.

또한 CSV 파일 저장 기능을 통해 프로그램 종료 후에도 식재료 데이터가 유지되며, 입력 예외 처리와 사용자 안내창을 통해 프로그램의 안정성과 사용성을 높였다.

이를 통해 사용자는 냉장고 속 식재료를 체계적으로 관리하고, 유통기한이 임박한 재료를 우선적으로 소비하며, 보유 재료를 활용한 메뉴를 추천받을 수 있다.
