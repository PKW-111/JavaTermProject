# 7. GitHub Release 배포

## 7.1 문서 목적

본 문서는 FreshKeeper Smart Fridge의 완성된 버전을 GitHub Release로 배포하기 위한 절차와 릴리즈 정보를 정리한 문서이다.

Release를 생성하면 교수와 팀원이 특정 시점의 완성된 소스 코드를 쉽게 확인하고 다운로드할 수 있다.

---

## 7.2 배포 대상

| 항목       | 내용                                                 |
| -------- | -------------------------------------------------- |
| 프로젝트명    | FreshKeeper Smart Fridge                           |
| 배포 버전    | `v1.0.0`                                           |
| 대상 브랜치   | `main`                                             |
| 배포 유형    | 최초 완성 버전                                           |
| 배포 상태    | 최종 테스트 완료 후 생성                                     |
| 주요 실행 방식 | IntelliJ에서 `Main.java` 또는 `FreshKeeperGUI.java` 실행 |

---

## 7.3 버전 관리 기준

본 프로젝트는 다음과 같은 버전 형식을 사용한다.

```text
v메이저.마이너.패치
```

| 버전       | 의미                  |
| -------- | ------------------- |
| `v1.0.0` | 최초 완성 버전            |
| `v1.1.0` | 새로운 기능 추가           |
| `v1.1.1` | 기존 기능의 오류 수정        |
| `v2.0.0` | 프로그램 구조나 기능의 대규모 변경 |

FreshKeeper Smart Fridge의 최초 최종 제출 버전은 `v1.0.0`으로 지정한다.

---

## 7.4 Release 생성 절차

1. GitHub 저장소의 `main` 브랜치에 최종 코드를 Push한다.
2. 저장소 메인 화면의 `Releases` 메뉴로 이동한다.
3. `Draft a new release` 또는 `Create a new release`를 선택한다.
4. 태그에 `v1.0.0`을 입력한다.
5. 대상 브랜치를 `main`으로 설정한다.
6. Release 제목과 설명을 작성한다.
7. 필요한 경우 프로젝트 ZIP 또는 JAR 파일을 첨부한다.
8. `Publish release`를 눌러 배포한다.

## 7.6 Release Note

```md
# v1.0.0 - FreshKeeper Smart Fridge 최종 완성

## 주요 기능

- Java Swing 기반 스마트 냉장고 GUI
- 식재료 등록, 조회, 검색, 수정, 삭제
- 유통기한 남은 일수 계산
- 만료, 위험, 주의, 안전 상태 분류
- 상태별 식재료 개수 표시
- 유통기한 임박 및 만료 알림
- 보유 식재료 기반 메뉴 추천
- 보유 재료와 부족 재료 표시
- 메뉴 추천 점수 계산
- 추천 메뉴 조리 기능
- 조리 후 식재료 수량 자동 차감
- CSV 데이터 저장 및 불러오기
- 잘못된 수량 및 날짜 입력 예외 처리

## 해결한 오류

- GUI 한글이 네모로 표시되는 문제 해결
- 입력칸 안내 문구가 부족한 문제 개선
- 검색 결과 중복 표시 문제 해결
- 조리 후 추천 화면 갱신 문제 해결
- 상태 카드 숫자가 갱신되지 않는 문제 해결

## 팀원

- 조우림
```

## 7.7 배포 전 확인 항목

* [ ] `main` 브랜치에 최종 코드가 반영되어 있는가?
* [ ] 프로그램이 정상적으로 실행되는가?
* [ ] 식재료 등록, 수정, 삭제 기능이 정상 동작하는가?
* [ ] 상태 카드 숫자가 테이블과 일치하는가?
* [ ] 메뉴 추천과 조리 기능이 정상 동작하는가?
* [ ] CSV 저장 후 재실행 시 데이터가 복원되는가?
* [ ] README와 docs 내용이 실제 코드와 일치하는가?
* [ ] `out/` 폴더가 GitHub에서 제외되어 있는가?
* [ ] Release 태그가 `v1.0.0`으로 작성되어 있는가?
