# 4-summary.md 작성 형식

## 4.1 구현 현황 요약

| 구분 | 수량 |
|---|---:|
| 총 요구사항 수 | 9개 |
| ☑ 완성 | 4개 |

| 구분 | 수량 |
|---|---:|
| ◇ 부분완성 | 3개 |
| ✖ 미완성 | 2개 |
| 구현율 (완성 기준) | 44% |

> 구현율 = (완성 수 / 총 요구사항 수) × 100

### 구현 현황 메모

- 완성: 유통기한 남은 일수 계산, 상태 분류, 메뉴 추천, 부족 재료 표시
- 부분완성: 식재료 등록/조회/삭제의 핵심 로직은 존재하나 콘솔 UI 연동이 더 필요함
- 미완성: 입력 예외 처리, 데이터 저장/불러오기

## 4.2 프로젝트 회고

### 잘 된 점

- 기능을 식재료 관리, 유통기한 처리, 메뉴 추천으로 나눠 구조를 이해하기 쉽게 정리했다.
- `Food`, `Recipe`, `RecipeResult`처럼 역할이 다른 데이터를 분리해 두어 확장하기 쉬운 형태로 만들었다.
- 유통기한 상태를 기준별로 나눠 출력하도록 설계해 사용자가 직관적으로 확인할 수 있게 했다.

### 아쉬운 점 / 개선 사항

- 콘솔 입력 처리와 저장/불러오기 기능이 아직 완전히 연결되지 않아 프로그램 흐름이 매끄럽지 않다.
- 추천 결과와 식재료 목록 출력 형식을 더 정리하면 가독성이 좋아질 것이다.
- 다음 단계에서는 예외 처리와 파일 저장을 먼저 마무리하고, 이후 화면 흐름을 통합하는 것이 필요하다.

### 팀원 기여 (팀 프로젝트의 경우)

| 이름 | 담당 요구사항 | 기여 내용 요약 |
|---|---|---|
| 박경우 | F-01, F-02, F-03, F-04, F-05, F-06, F-07 | 식재료 관리와 추천 로직의 핵심 구조를 담당 |
| 조우림 | F-08, F-09 | 입력 예외 처리, 저장/불러오기, 문서 정리 및 안정화 담당 |
