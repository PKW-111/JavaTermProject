    # 4. 기능구현 및 단계별 구현내용

FreshKeeper는 **사용자가 직접 입력한 식재료 정보를 관리하고, 유통기한 상태를 확인하며, 보유 재료를 기반으로 메뉴를 추천하는 Java 콘솔 프로그램**이다.  
이 문서는 프로젝트 개요와 요구사항을 바탕으로 실제 구현한 기능을 기능별/단계별로 정리한 내용이다.

---

## 4.1 기능구현 개요

| 기능 | 구현 내용 | 관련 클래스 |
|---|---|---|
| 식재료 등록 | 이름, 카테고리, 수량, 보관 위치, 유통기한을 입력받아 식재료를 추가한다. | `Main`, `FoodService`, `Food` |
| 식재료 전체 조회 | 등록된 식재료 목록을 표 형태로 출력한다. | `Main`, `Food` |
| 식재료 삭제 | 식재료 ID를 기준으로 삭제하고 저장 파일을 갱신한다. | `Main`, `FoodService` |
| 유통기한 계산 | 현재 날짜와 유통기한을 비교하여 남은 일수를 계산한다. | `Food` |
| 상태 분류 | 남은 일수에 따라 안전/주의/위험/오늘까지/만료로 분류한다. | `Food` |
| 메뉴 추천 | 보유 식재료와 레시피를 비교해 만들 수 있는 메뉴를 추천한다. | `Main`, `RecipeService`, `Recipe`, `RecipeResult` |
| 부족 재료 표시 | 추천 메뉴에 필요한 재료 중 부족한 항목을 함께 출력한다. | `RecipeService`, `RecipeResult` |
| 입력 예외 처리 | 숫자, 날짜, 빈 값 입력 오류를 처리해 프로그램이 종료되지 않도록 한다. | `Main` |
| 데이터 저장/불러오기 | 식재료 정보를 텍스트 파일로 저장하고 실행 시 다시 불러온다. | `FoodService` |

---

## 4.2 단계별 구현 내용

### 1단계: 프로그램 기본 구조 구현

- `Main` 클래스를 중심으로 콘솔 메뉴 구조를 먼저 구현하였다.
- `while` 반복문과 `switch` 문을 사용해 기능 선택 흐름을 구성하였다.
- 사용자는 메뉴 번호를 입력하여 식재료 등록, 조회, 삭제, 메뉴 추천, 종료 기능을 실행한다.

### 2단계: 식재료 데이터 모델 구현

- `Food` 클래스를 통해 식재료의 기본 정보를 저장하도록 구현하였다.
- 저장 항목은 `id`, `name`, `category`, `quantity`, `storageLocation`, `expiryDate`이다.
- `getDaysLeft()` 메소드로 현재 날짜 기준 남은 일수를 계산하고, `getStatus()` 메소드로 상태를 분류한다.

### 3단계: 식재료 등록 및 관리 기능 구현

- `FoodService`에서 식재료 목록을 `ArrayList`로 관리하도록 구현하였다.
- 식재료 등록 시 고유 ID를 자동으로 부여하고, 유통기한이 빠른 순으로 정렬되도록 처리하였다.
- 식재료 조회는 등록된 전체 목록을 한 번에 확인할 수 있도록 표 형식 출력으로 구현하였다.
- 식재료 삭제는 ID를 기준으로 대상 식재료를 찾아 제거한 뒤 저장 파일을 다시 갱신한다.

### 4단계: 입력 예외 처리 구현

- 수량과 메뉴 번호는 정수 입력만 받도록 처리하였다.
- 날짜는 `yyyy-MM-dd` 형식으로 입력받고, 형식이 맞지 않으면 다시 입력하도록 구현하였다.
- 빈 문자열 입력을 막아 필수 정보를 누락하지 않도록 처리하였다.

### 5단계: 데이터 저장 및 불러오기 기능 구현

- 프로그램 종료 후에도 식재료 정보가 유지되도록 텍스트 파일 기반 저장 기능을 구현하였다.
- 실행 시 기존 저장 파일이 있으면 자동으로 불러오고, 없으면 새 목록으로 시작한다.
- 식재료 추가/삭제 후에는 즉시 저장하여 데이터 불일치를 방지한다.

### 6단계: 레시피 및 메뉴 추천 기능 구현

- `Recipe` 클래스로 메뉴 이름과 필요한 재료 목록을 관리하도록 구현하였다.
- `RecipeService`에서 보유 식재료와 레시피를 비교하여 추천 점수를 계산한다.
- 보유 재료가 많을수록 점수가 높아지도록 구성해 추천 우선순위를 정렬하였다.
- `RecipeResult`에는 추천 메뉴명, 보유 재료, 부족 재료, 점수를 함께 저장한다.

### 7단계: 최종 출력 및 사용자 경험 개선

- 추천 결과를 순위별로 출력하여 사용자가 우선적으로 만들 수 있는 메뉴를 쉽게 확인할 수 있도록 하였다.
- 보유 재료와 부족 재료를 분리 출력해 장보기 여부를 판단하기 쉽게 구성하였다.
- 전체 기능을 콘솔 화면 흐름에 맞게 연결하여 하나의 완성된 프로그램으로 통합하였다.

## 4.3 기능별 구현 포인트

### 식재료 등록

- 사용자 입력값을 검증한 뒤 `Food` 객체를 생성한다.
- 등록 직후 저장 파일에 반영된다.

### 식재료 조회

- 식재료 ID, 이름, 카테고리, 수량, 보관 위치, 유통기한, 남은 일수, 상태를 함께 출력한다.
- 유통기한이 임박한 식재료를 한눈에 확인할 수 있도록 정렬 기준을 적용하였다.

### 식재료 삭제

- 삭제 전 선택된 식재료 정보를 다시 보여주어 실수 삭제를 줄이도록 구성하였다.
- 삭제 확인 후에만 실제 목록에서 제거한다.

### 유통기한 상태 분류

- 남은 일수 기준으로 다음과 같이 분류한다.

| 남은 일수 | 상태 |
|---|---|
| 8일 초과 | 안전 |
| 3~7일 | 주의 |
| 1~2일 | 위험 |
| 0일 | 오늘까지 |
| 0일 미만 | 만료 |

### 메뉴 추천

- 현재 보유한 식재료 이름과 수량을 기준으로 레시피와 비교한다.
- 부족한 재료도 함께 출력하여 실제 조리 가능 여부를 판단할 수 있도록 한다.

---

## 4.4 구현 결과 요약

FreshKeeper는 식재료 등록/조회/삭제, 유통기한 계산, 상태 분류, 메뉴 추천, 부족 재료 표시, 파일 저장 기능을 하나의 콘솔 프로그램으로 통합하여 구현하였다.  
이를 통해 사용자는 냉장고 속 식재료를 체계적으로 관리하고, 유통기한이 임박한 재료를 우선 활용할 수 있다.
