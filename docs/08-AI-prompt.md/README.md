# 8. GitHub Collaborator 등록

## 8.1 문서 목적

본 문서는 FreshKeeper Smart Fridge 저장소의 팀원과 평가자 접근 권한을 관리하고, Collaborator 등록 상태를 기록하기 위해 작성하였다.

---

## 8.2 Collaborator 구성

| 구분       | GitHub 계정    | 권한    | 목적                          |
| -------- | ------------ | ----- | --------------------------- |
| 저장소 관리자  | PKW-111      | Admin | 저장소 설정, 코드 통합, 최종 제출 관리     |
| 담당 교수    | andy-yun     | Read  | 프로젝트 코드와 문서 확인 및 평가         |
| 공동 작업 팀원 | 팀원 GitHub 계정 | Write | 기능 구현, 문서 작성, Commit 및 Push |

> 팀원 GitHub 계정은 실제 초대한 계정명으로 변경한다.

---

## 8.3 등록 절차

1. GitHub에서 FreshKeeper 저장소를 연다.
2. 상단의 `Settings`를 선택한다.
3. 왼쪽 메뉴에서 `Collaborators`를 선택한다.
4. `Add people` 버튼을 누른다.
5. 교수 또는 팀원의 GitHub 아이디나 이메일을 입력한다.
6. 필요한 권한을 선택한 후 초대를 보낸다.
7. 상대방이 초대를 수락했는지 확인한다.

---

## 8.4 권한 기준

| 대상       | 권장 권한 | 가능한 작업                             |
| -------- | ----- | ---------------------------------- |
| 담당 교수    | Read  | 코드, 문서, Issue, Pull Request 확인     |
| 공동 작업 팀원 | Write | 코드 수정, Commit, Push, Issue 및 PR 작업 |
| 팀장       | Admin | 저장소 설정, Collaborator 관리, 최종 통합     |

교수에게는 프로젝트 평가에 필요한 읽기 권한만 제공하고, 저장소 설정 변경 권한은 부여하지 않는다.

팀원에게는 프로젝트 파일을 직접 수정하고 Push할 수 있도록 작업 권한을 제공한다.

---

## 8.5 등록 확인

Collaborator 등록 여부는 다음 경로에서 확인한다.

```text
GitHub 저장소
→ Settings
→ Collaborators
→ Manage access
```

| 확인 항목            | 상태                    |
| ---------------- | --------------------- |
| 교수 `andy-yun` 초대 | 완료                    |
| 교수 초대 수락 여부      | Collaborators 목록에서 확인 |
| 공동 작업 팀원 초대      | 완료                    |
| 팀원 초대 수락 여부      | Collaborators 목록에서 확인 |
| 저장소 접근 가능 여부     | 확인 완료                 |
| 권한 수준            | 교수 Read / 팀원 Write 권장 |

---

## 8.6 주의사항

* GitHub 아이디가 정확한지 확인한 후 초대한다.
* 초대가 `Pending invite` 상태이면 아직 상대방이 수락하지 않은 것이다.
* 교수에게 Admin 권한을 부여하지 않는다.
* 팀원이 아닌 사용자를 불필요하게 등록하지 않는다.
* 프로젝트 종료 후에는 필요에 따라 Collaborator 권한을 제거할 수 있다.
* 저장소가 Public이어도 평가와 협업 관리를 위해 Collaborator 등록 상태를 유지한다.
