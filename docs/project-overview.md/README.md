# 1. 프로젝트 개요

## 1.1 프로젝트명

**FreshKeeper Smart Fridge: 사용자 입력 기반 냉장고 유통기한 관리 및 메뉴 추천 프로그램**

---

## 1.2 프로젝트 소개

FreshKeeper Smart Fridge는 사용자가 직접 입력한 식재료 정보를 기반으로 냉장고 속 식재료의 수량과 유통기한을 관리하는 Java 데스크톱 프로그램이다.

사용자는 식재료의 이름, 카테고리, 수량, 보관 위치, 유통기한을 등록할 수 있다. 프로그램은 현재 날짜를 기준으로 유통기한까지 남은 일수를 자동으로 계산하고, 계산 결과에 따라 식재료를 안전, 주의, 위험, 오늘까지, 만료 상태로 분류한다.

등록된 식재료는 검색, 수정, 삭제할 수 있으며, 만료되었거나 유통기한이 임박한 식재료는 알림창과 상태 요약 카드를 통해 확인할 수 있다.

또한 보유 중인 식재료와 레시피 데이터를 비교하여 만들 수 있는 메뉴를 추천한다. 추천 결과에는 보유 재료, 부족 재료, 추천 점수가 함께 표시되며, 사용자는 추천된 메뉴를 선택하여 조리할 수 있다. 메뉴를 조리하면 사용된 식재료의 수량이 자동으로 차감된다.

초기에는 콘솔 기반 프로그램으로 구현하였으나, 사용성과 시각적 완성도를 높이기 위해 Java Swing을 이용한 스마트 냉장고 디스플레이 형태의 GUI 프로그램으로 확장하였다.

식재료 데이터는 CSV 파일에 저장되며, 프로그램을 종료하고 다시 실행해도 기존 데이터가 유지된다.

---

## 1.3 프로젝트 목적

본 프로젝트의 목적은 냉장고 속 식재료를 효율적으로 관리하고, 유통기한이 임박한 재료를 우선적으로 사용할 수 있도록 돕는 것이다.

일상생활에서 사용자는 냉장고에 어떤 재료가 있는지 잊어버리거나, 유통기한을 확인하지 못해 식재료를 버리는 경우가 있다. 또한 식재료를 메모해 두더라도 수량 변화와 유통기한을 지속해서 관리하기 어렵다.

FreshKeeper Smart Fridge는 사용자가 구매한 식재료를 직접 등록하고, 전체 조회와 검색을 통해 현재 보유 중인 재료를 확인할 수 있도록 한다. 유통기한 상태를 자동으로 계산하고 시각적으로 표시하여 어떤 재료를 먼저 사용해야 하는지 쉽게 판단할 수 있도록 한다.

또한 보유 재료 기반 메뉴 추천과 조리 후 수량 차감 기능을 통해 단순한 식재료 목록 관리에서 나아가 실제 냉장고 재고 관리 흐름을 구현하는 것을 목표로 한다.

---

## 1.4 개발 배경

현대 사회에서는 자취생이나 바쁜 대학생처럼 직접 식재료를 구매하고 관리해야 하는 사람들이 많다. 그러나 냉장고 속 재료를 매번 직접 확인하는 것은 번거롭고, 유통기한이 짧은 재료를 제때 사용하지 못하는 경우가 자주 발생한다.

기존 방식은 냉장고를 직접 열어 확인하거나 메모장에 식재료를 기록하는 방식이 대부분이다. 하지만 이러한 방법은 유통기한이 얼마나 남았는지 한눈에 파악하기 어렵고, 식재료를 사용한 이후 변경된 수량을 관리하기 어렵다는 한계가 있다.

또한 보유한 재료로 어떤 음식을 만들 수 있는지 사용자가 직접 판단해야 하며, 부족한 재료를 별도로 확인해야 하는 불편함이 있다.

따라서 본 프로젝트에서는 사용자가 직접 입력한 식재료 정보를 바탕으로 다음 기능을 제공하는 프로그램을 개발하였다.

* 식재료 등록, 조회, 검색, 수정, 삭제
* 유통기한 자동 계산 및 상태 분류
* 유통기한 임박 및 만료 알림
* 보유 재료 기반 메뉴 추천
* 추천 메뉴 조리 후 식재료 수량 차감
* CSV 파일을 이용한 데이터 저장 및 불러오기
* Java Swing 기반 스마트 냉장고 GUI

## 1.5 대상 사용자

* 냉장고 속 식재료를 체계적으로 관리하고 싶은 사용자
* 자취하며 식재료를 직접 관리하는 대학생
* 유통기한이 임박한 재료를 먼저 사용하고 싶은 사용자
* 보유 재료로 만들 수 있는 메뉴를 추천받고 싶은 사용자
* 같은 식재료를 중복 구매하는 일을 줄이고 싶은 사용자
* 프로그램을 종료한 후에도 식재료 기록을 유지하고 싶은 사용자
* 식재료 사용 후 남은 수량을 지속해서 관리하고 싶은 사용자

| 부족 재료 확인 어려움      | 장보기 계획이 불편함           | 추천 메뉴의 부족 재료 표시              |
| 식재료 사용 후 수량 직접 변경 | 재고 수량을 계속 수정해야 함      | 메뉴 조리 후 사용된 식재료 수량 자동 차감     |
| 텍스트 중심 콘솔 화면      | 사용 방법을 직관적으로 파악하기 어려움 | Java Swing 기반 스마트 냉장고 GUI 제공 |

## 1.7 주요 기능

| 기능        | 설명                                          |
| --------- | ------------------------------------------- |
| 식재료 등록    | 식재료 이름, 카테고리, 수량, 보관 위치, 유통기한을 입력한다.        |
| 식재료 전체 조회 | 등록된 식재료의 정보와 유통기한 상태를 테이블에서 확인한다.           |
| 식재료 검색    | 이름, 카테고리, 보관 위치를 기준으로 원하는 식재료를 검색한다.        |
| 식재료 수정    | 선택한 식재료의 이름, 카테고리, 수량, 위치, 유통기한을 수정한다.      |
| 식재료 삭제    | 목록에서 선택한 식재료를 확인 후 삭제한다.                    |
| 유통기한 계산   | 현재 날짜를 기준으로 유통기한까지 남은 일수를 계산한다.             |
| 위험도 분류    | 남은 날짜에 따라 안전, 주의, 위험, 오늘까지, 만료로 분류한다.       |
| 상태 요약 카드  | 만료, 위험, 주의, 안전 상태의 식재료 개수를 상단 카드에 표시한다.     |
| 유통기한 알림   | 만료되었거나 유통기한이 임박한 식재료를 알림창으로 표시한다.           |
| 메뉴 추천     | 보유 중인 식재료를 기반으로 만들 수 있는 메뉴를 추천한다.           |
| 추천 점수 계산  | 보유 재료, 부족 재료, 임박 재료 여부에 따라 추천 점수를 계산한다.     |
| 부족 재료 표시  | 추천 메뉴에 필요한 재료 중 보유하지 않은 재료를 표시한다.           |
| 메뉴 조리     | 추천된 메뉴 중 하나를 선택하여 조리할 수 있다.                 |
| 수량 자동 차감  | 메뉴를 조리하면 사용된 식재료의 수량을 자동으로 차감한다.            |
| 수량 0 처리   | 수량이 0이 된 식재료의 삭제 여부를 사용자에게 확인한다.            |
| CSV 저장    | 현재 식재료 목록을 CSV 파일에 저장한다.                    |
| CSV 불러오기  | 프로그램 실행 시 CSV 파일에서 기존 식재료 정보를 불러온다.         |
| 입력 예외 처리  | 잘못된 수량이나 날짜를 입력해도 프로그램이 종료되지 않고 안내한다.       |
| Swing GUI | 스마트 냉장고 화면을 참고한 Java Swing 기반 사용자 화면을 제공한다. |

## 1.8 프로젝트 범위

### 포함 범위

* 사용자가 직접 식재료 정보 입력
* 식재료 등록, 전체 조회, 검색, 수정, 삭제
* 유통기한 남은 일수 계산
* 안전, 주의, 위험, 오늘까지, 만료 상태 분류
* 상태별 식재료 개수 표시
* 유통기한 임박 및 만료 알림
* 보유 재료 기반 메뉴 추천
* 추천 점수 계산
* 부족 재료 표시
* 추천 메뉴 조리
* 조리 후 식재료 수량 자동 차감
* 수량 0인 식재료 삭제 확인
* CSV 파일 저장 및 불러오기
* Java Swing 기반 GUI 프로그램 구현
* 잘못된 입력에 대한 예외 처리

### 제외 범위

* 실제 냉장고 내부 자동 스캔
* 바코드 스캔
* 카메라 이미지 인식
* IoT 냉장고 센서 연동
* 외부 레시피 API 연동
* 웹 서버 및 클라우드 동기화
* 모바일 앱 개발
* 회원가입 및 로그인 기능
* 원격 알림 전송
* 데이터베이스 연동

본 프로젝트는 실제 냉장고 내부를 자동으로 인식하는 시스템이 아니라, 사용자가 구매한 식재료를 직접 입력하고 관리하는 Java 데스크톱 프로그램이다.

식재료 데이터는 별도의 데이터베이스 대신 CSV 파일을 이용하여 관리한다.

## 1.9 기대 효과

* 냉장고 속 식재료를 체계적으로 관리할 수 있다.
* 식재료를 이름, 카테고리, 보관 위치로 빠르게 검색할 수 있다.
* 잘못 입력한 식재료 정보를 삭제하지 않고 수정할 수 있다.
* 유통기한이 임박하거나 지난 식재료를 쉽게 확인할 수 있다.
* 어떤 재료를 먼저 사용해야 하는지 판단할 수 있다.
* 상태 요약 카드를 통해 냉장고의 식재료 상태를 한눈에 확인할 수 있다.
* 보유 재료 기반 메뉴 추천을 통해 식재료 활용도를 높일 수 있다.
* 부족한 재료를 확인하여 장보기 계획에 도움을 받을 수 있다.
* 메뉴 조리 후 식재료 수량이 자동으로 변경되어 재고 관리가 편리해진다.
* 프로그램을 종료하고 다시 실행해도 입력한 식재료 데이터가 유지된다.
* 음식물 낭비를 줄이는 데 도움을 줄 수 있다.
* Java의 객체지향 설계와 GUI 프로그래밍 개념을 실제 프로젝트에 적용할 수 있다.
* 파일 입출력, 예외 처리, 날짜 계산, 컬렉션, 이벤트 처리 기술을 함께 학습할 수 있다.
