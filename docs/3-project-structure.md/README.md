# 3. 프로젝트 구조 및 역할분담 · 일정관리

이 문서는 FreshKeeper의 **구조 설계, 역할 분담, 주차 기반 일정 계획**을 평가 기준에 맞춰 정리한 요약 페이지다.  
상세 내용은 아래 본문 문서에서 확인한다.

- [Project_Architecture_and_Schedule.md](../Project_Architecture_and_Schedule.md/Project_Architecture_and_Schedule.md)

## 요약

| 항목 | 핵심 내용 |
|---|---|
| 프로젝트 구조 | 요구사항(F-01~F-09) 기반으로 기능-모듈 매핑, 화면 흐름, 시스템 구조도(Mermaid) 명세 |
| 역할 분담 | 박경우: 핵심 기능/통합 리드, 조우림: 안정화/테스트/문서 리드 + RACI 기준 책임 분리 |
| 협업 방식 | 기능 브랜치 + PR 리뷰, 주 2회 점검, 이슈 기반 작업 관리 |
| 일정 계획 | 1주차 프로젝트 개요 → 2주차 요구사항 분석 및 설계 → 3주차 프로젝트 구조 및 역할 일정 관리 계획 → 4주차 통합 안정화 → 5주차 제출 준비 → 6주차 최종 제출/발표 대응 |
| 중간 점검 | M1~M8 마일스톤으로 설계 동결, 기능 단계 점검, 통합/품질 확인, 최종 제출 관리 |
| 리스크 대응 | 지연/충돌/예외 누락/데이터 불일치 리스크에 대해 Trigger 기반 대응 방안 명시 |

## 평가 포인트 대응

| 평가 항목 | 대응 방식 |
|---|---|
| 구조의 명확성 | 요구사항-기능-모듈 추적표 + 구조도 + 시퀀스 명세 |
| 역할 분담의 적절성 | 팀원별 책임 범위와 완료 기준 명확화 |
| 일정 계획의 현실성 | 6주 주차 계획 + 마일스톤 + WBS 제시 |
| 문서 완성도 | 표 중심 구성, 용어 통일, 점검 기준/완료 기준 포함 |
